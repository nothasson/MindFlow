## 一、LLM 自动提取知识图谱的最佳实践

### 最新研究结论

1. **EDC 框架（EMNLP 2024）**：Extract-Define-Canonicalize 三步法
2. **Prompt 格式比模型选择更重要**
3. **布鲁姆认知分类法可用于 LLM 自动分级**（2025 Elsevier 论文）

### 建议的4级粒度体系

- **L1（学科级）**：线性代数、微积分
- **L2（章节级）**：矩阵运算、特征值分解
- **L3（知识点级）**：矩阵乘法、矩阵求逆
- **L4（技能级）**：2x2矩阵求逆、3x3行列式计算

### 当前问题

extractor.py 的 Prompt "提取 3-8 个核心知识点" 过于模糊，无粒度标准、无布鲁姆分类、关系类型单一、无验证机制、长文档直接截断。

### 优化建议

**P0：升级提取 Prompt**（每个知识点包含 concept、description、bloom_level、importance，关系包含 prerequisite/similar/application/part_of/causal + strength）

**P1：分块提取+合并去重**（按章节分块 3000-5000 tokens，独立提取后 LLM 合并去重）

**P1：提取质量验证**（JSON 校验 + LLM 交叉验证 + 用户反馈修正）

## 二、知识图谱的关系类型设计

### 推荐的关系类型（按价值排序）

| 关系类型 | 说明 | 对学习的价值 |
|---------|------|------------|
| prerequisite | 前置依赖 | 学习路径规划、错误根源追踪 |
| similar | 相似/易混淆 | 对比学习、易错题生成 |
| part_of | 包含关系 | 知识结构层次化 |
| application | 应用场景 | 连接理论与实践 |
| causal | 因果关系 | 深层理解 |
| generalization | 特例/推广 | 归纳演绎思维 |
| related | 弱关联 | 知识拓展 |

### 前端可视化建议
- prerequisite：实线箭头
- similar：红色虚线双向（标注"易混淆"）
- part_of：灰色虚线
- application：绿色虚线

**P0：`similar` 关系是最有价值的新增**（直接支持对比学习和错题变式题）

## 三、基于知识图谱的个性化学习路径规划

### 核心算法

```
输入：知识图谱 G、学生掌握度 M、学习目标 T
算法：
1. 拓扑排序：找出目标 T 的所有前置依赖链
2. 过滤已掌握：去掉 confidence > 0.8 的节点
3. 优先级排序：按距离目标的最短路径排序
4. 输出路径：[前置1, 前置2, ..., 目标T]
```

### CurriculumAgent 结合图谱数据

在 Prompt 中注入：
```
当前学生状态：
- 已掌握：[概念A(95%), 概念B(80%)]
- 薄弱点：[概念C(30%), 概念D(20%)]
- 概念C 的前置依赖：概念E(未学习), 概念F(60%)
- 建议路径：概念E → 概念F(复习) → 概念C → 概念D
```

**P1：基于拓扑排序的学习路径生成**

## 四、错误根源追踪

### 核心算法

```
当学生在知识点 X 出错时：
1. 获取 X 的所有前置知识 [P1, P2, P3, ...]
2. 检查每个前置知识的掌握度
3. confidence < 0.5 → 标记为"可能的根源"
4. 递归检查，找最深层薄弱点
5. 最深层薄弱点 = 根本原因
```

### DiagnosticAgent 增加根源分析

```json
{
  "root_cause": {
    "weak_prerequisites": [
      {"concept": "线性变换", "confidence": 0.3, "depth": 2}
    ],
    "suggestion": "建议先复习'线性变换'概念"
  }
}
```

**P1：松鼠 AI 核心专利方向，用图的递归遍历实现简化版**

## 五、知识点向量化

### 方案

- Embedding 输入："{concept}: {description}。认知层级: {bloom_level}。"
- 模型推荐：bge-small-zh-v1.5（中文最佳，512维）
- 存储：Qdrant（已配置但未使用）

### 向量驱动的功能
1. 相似概念推荐
2. 易混淆概念自动检测（相似度 > 0.85）
3. 跨资料关联
4. 语义搜索

**P2：Qdrant 已配置但代码中无调用，向量化后解锁语义搜索和自动发现易混淆概念**

## 六、优先级汇总

| 优化项 | 优先级 | 实现难度 | 建议工期 |
|--------|--------|---------|---------|
| 升级提取 Prompt（粒度+布鲁姆+多关系） | P0 | 低 | 1-2天 |
| 扩展关系类型（especially similar） | P0 | 低 | 1天 |
| 分块提取+合并去重 | P1 | 中 | 2-3天 |
| 前置知识薄弱检测（根源追踪） | P1 | 中 | 2-3天 |
| 基于拓扑排序的学习路径 | P1 | 中 | 2-3天 |
| DiagnosticAgent 结合图谱 | P1 | 低 | 1-2天 |
| 知识点向量化（Qdrant） | P2 | 中 | 3-4天 |

### 核心结论
1. 知识点提取质量是整个系统的基础
2. `similar` 关系是最有价值的新增
3. 错误根源追踪是松鼠 AI 核心方向
4. 当前规模不需要图数据库，PostgreSQL 递归 CTE 够用
