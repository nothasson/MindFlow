# MindFlow 可优化功能点清单（个人学习工具方向）

> **任务**：综合 5 份竞品调研和技术分析报告，以"个人学习工具（非社交）"为核心定位，输出按优先级排序的可优化功能点清单
> **执行时间**：2026年4月10日
> **定位**：个人学习工具，不做社交功能

---

## P0 — 必须做（核心体验 + 差异化）

### 1. FSRS 算法迁移（替换 SM-2）
- **现状**：SM-2（1987年）算法过时，固定参数无法个性化
- **方案**：引入 `go-fsrs` v4 官方 Go 包，21 个可优化权重
- **效果**：复习效率提升 20-30%，同等保留率下减少日复习量
- **工作量**：2-3 天（含数据库 migration + 前端评分 UI 改为 Again/Hard/Good/Easy 四按钮）
- **来源**：07-FSRS 调研

### 2. 苏格拉底对话 Prompt 升级
- **现状**：Prompt 过于笼统，缺少具体教学动作指导，无错误场景示范
- **方案**：引入 SocraticLM 的 4 项教学能力（IARA/CARA/SER/SRR）+ ConvoLearn 脚手架框架 + 递进式引导策略
- **效果**：教学质量预期提升 10-15%
- **工作量**：2-3 天
- **来源**：08-Prompt 工程调研

### 3. 错误诊断精细化（5+3 分类体系）
- **现状**：仅 3 种错误类型（概念错/方法错/粗心），过于粗糙
- **方案**：升级为 5 种基础错误（知识遗漏/概念混淆/概念错误/方法错误/计算错误）+ 3 种元认知错误（过度自信/策略错误/表述不清），每种错误映射到专属引导策略
- **效果**：诊断精准度大幅提升，针对性教学
- **工作量**：2-3 天
- **来源**：08-Prompt 工程调研

### 4. 提示词注入防护
- **现状**：用户输入直接进 Prompt，无防护，攻击成功率 76-82%
- **方案**：四层防御 — ① Prompt 三明治防御 ② 输出检测（是否包含直接答案）③ 角色隔离声明 ④ Few-shot 攻击示例防御
- **效果**：防止学生通过 prompt 注入让 AI 直接给答案
- **工作量**：1 天
- **来源**：08-Prompt 工程调研

### 5. 知识点提取 Prompt 升级
- **现状**："提取 3-8 个知识点" 过于模糊，无粒度标准、无布鲁姆分类、关系类型仅 prerequisite
- **方案**：4 级粒度体系（L1-L4）+ 布鲁姆认知层级标注 + 重要性权重 + 扩展关系类型（prerequisite/similar/application/part_of/causal + strength）
- **效果**：知识图谱质量提升 50%+，影响全链路
- **工作量**：1-2 天
- **来源**：09-知识图谱调研

### 6. 错题变式题系统
- **现状**：无错题变式题功能
- **方案**：6 种变式类型 — 参数变换、情境变换、角度变换、反向出题、简化变式、综合变式。与 Diagnostic Agent 联动（诊断结果驱动变式类型选择），变式题纳入 FSRS 复习队列
- **效果**：核心教学差异化，针对性突破错误根源
- **工作量**：3-4 天
- **来源**：07-FSRS 调研 + 08-Prompt 调研


### 7. 错题自动收集
- **现状**：无错题本功能
- **方案**：测验答错 → 自动入错题本；对话中诊断出薄弱点 → 自动标记。零手动操作
- **效果**：零摩擦错题管理，配合变式题系统形成闭环
- **工作量**：2 天
- **来源**：10-体验设计调研

## P1 — 重要（体验提升 + 智能化）

### 9. Bloom 分类法出题
- **方案**：QuizAgent 按学生掌握度自动匹配 Bloom 层级（记忆→理解→应用→分析→评价→创造），每次出题覆盖不同层级
- **工作量**：2 天
- **来源**：08-Prompt 调研

### 10. AI 晨间简报（今日学习计划）
- **方案**：每次打开 app 显示 AI 推荐的学习计划（待复习项 + 新知识推荐 + 可选测验），每项附推荐理由 + 预估学习时间，支持"延后/跳过/替换"
- **工作量**：3 天
- **来源**：10-体验设计调研

### 11. 错误根源追踪（知识图谱）
- **方案**：学生答错时，通过知识图谱递归查找薄弱前置知识（类似松鼠 AI "追根溯源"），DiagnosticAgent 输出增加 root_cause 字段
- **工作量**：2-3 天
- **来源**：09-知识图谱调研

### 12. 基于拓扑排序的学习路径生成
- **方案**：拓扑排序找前置依赖链 → 过滤已掌握 → 按最短路径排序 → 注入 CurriculumAgent Prompt
- **工作量**：2-3 天
- **来源**：09-知识图谱调研

### 13. 学习仪表盘重设计
- **方案**：GitHub 贡献图样式热力图 + 掌握度环形图 + 薄弱知识点 Top 5（带行动按钮）+ 连续学习天数 + 今日 XP
- **工作量**：3-4 天
- **来源**：10-体验设计调研

### 14. 源文件引用锚定
- **方案**：AI 对话回复中嵌入 "[资料名:第X页]" 引用标记，用户可点击跳转原文（借鉴 NotebookLM）
- **工作量**：3 天
- **来源**：06-竞品对比

### 15. 上传后自动概览
- **方案**：资料上传解析后，自动生成摘要 + 3-5 个建议学习问题（借鉴 NotebookLM）
- **工作量**：1-2 天
- **来源**：06-竞品对比

### 16. 复习体验优化
- **方案**：多题型轮换（选择→填空→简答→判断）+ 进度条 + 连胜计数 + 默认 10 题/次 + 答错即时 AI 简短解释
- **工作量**：2-3 天
- **来源**：10-体验设计调研

### 17. 考试模式
- **方案**：用户设定考试日期 → AI 自动加速相关知识点复习频率 → 倒计时可视化 → 考后恢复常规
- **工作量**：2-3 天
- **来源**：10-体验设计调研

### 18. 分块提取 + 合并去重
- **方案**：长文档按章节分块（3000-5000 tokens），独立提取后 LLM 合并去重
- **工作量**：2-3 天
- **来源**：09-知识图谱调研

### 19. 对话式考察模式
- **方案**：Quiz Agent 增加"对话考察"模式，AI 和学生 5-10 分钟对话追问逻辑（借鉴 Khanmigo "AI 口头考试"）
- **工作量**：2 天
- **来源**：06-竞品对比

## P2 — 锦上添花（长期竞争力）

### 20. 知识点向量化（Qdrant）
- **方案**：知识点 embedding 存入 Qdrant，解锁语义搜索、相似概念推荐、易混淆概念自动检测
- **工作量**：3-4 天
- **来源**：09-知识图谱调研

### 21. 教学风格动态自适应
- **方案**：根据对话中的信号（错误率/卡住频率/回答深度）自动切换初学者/进阶/专家模式，无需手动 SetLevel()
- **工作量**：2-3 天
- **来源**：08-Prompt 调研

### 22. 多格式资料支持
- **方案**：扩展支持网页 URL 抓取、YouTube 字幕提取、纯文本粘贴（借鉴 NotebookLM）
- **工作量**：3-5 天
- **来源**：06-竞品对比

### 23. 知识图谱交互增强
- **方案**：节点大小=学习频率，节点颜色=掌握度，边粗细=关联强度，点击节点显示关联资料/对话/错题，高亮知识孤岛
- **工作量**：3-4 天
- **来源**：10-体验设计调研

### 24. 教学风格可选
- **方案**：让用户选择"深入原理"/"通俗比喻"/"考试导向"等教学风格（借鉴 NotebookLM Goals）
- **工作量**：1-2 天
- **来源**：06-竞品对比

### 25. 易混淆概念交错复习
- **方案**：知识图谱中 similar 关系的概念安排在同一复习 session 交错出现（LECTOR 算法启发）
- **工作量**：2 天
- **来源**：07-FSRS 调研

### 26. 资料全链路关联
- **方案**：资料→知识点→对话记录→测验题全链路关联，用户可从任何入口追溯完整学习脉络
- **工作量**：3-4 天
- **来源**：10-体验设计调研
