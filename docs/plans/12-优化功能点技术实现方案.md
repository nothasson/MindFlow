### 1. FSRS 算法迁移（替换 SM-2）

#### 现状代码分析

当前使用 SM-2 算法，实现在 `backend/internal/review/sm2.go`：

```go
// ReviewItem 复习项（SM-2 算法）
type ReviewItem struct {
    ConceptID      string
    EasinessFactor float64   // 固定初始值 2.5
    Interval       int       // 1 → 6 → EF * Interval
    Repetitions    int
    NextReview     time.Time
    LastReview     time.Time
    LastScore      int       // 0-5 六档评分
}
```

SM-2 的核心问题：
- **固定参数**：EF 初始值 2.5，所有知识点起步相同，无法个性化
- **评分粒度粗**：0-5 六档评分对用户不友好，前端没有直觉映射
- **无自适应**：不根据个人历史数据优化参数

```go
// backend/internal/review/fsrs.go
package review

import (
    "time"
    "github.com/open-spaced-repetition/go-fsrs/v3"
)

// Rating 映射（前端四按钮 → FSRS Rating）
// Again=1, Hard=2, Good=3, Easy=4
type Rating = fsrs.Rating

const (
    Again Rating = fsrs.Again  // 1 - 完全忘记
    Hard  Rating = fsrs.Hard   // 2 - 困难回忆
    Good  Rating = fsrs.Good   // 3 - 正常回忆
    Easy  Rating = fsrs.Easy   // 4 - 轻松回忆
)

// FSRSScheduler 封装 FSRS 调度逻辑
type FSRSScheduler struct {
    params fsrs.Parameters
}

func NewFSRSScheduler() *FSRSScheduler {
    p := fsrs.DefaultParameters()
    // 21 个可优化权重，后续可根据用户历史数据训练
    return &FSRSScheduler{params: p}
}

// Schedule 根据当前卡片状态和用户评分，计算下次复习时间
func (s *FSRSScheduler) Schedule(card fsrs.Card, rating Rating, now time.Time) fsrs.SchedulingInfo {
    scheduler := fsrs.NewFSRS(s.params)
    result := scheduler.Repeat(card, now)
    return result[rating]
}

// RatingToConfidence 评分映射到置信度
func RatingToConfidence(rating Rating) float64 {
    switch rating {
    case Easy:
        return 1.0
    case Good:
        return 0.85
    case Hard:
        return 0.55
    case Again:
        return 0.2
    default:
        return 0.0
    }
}
```

```go
## 核心原则
绝不直接给出答案。你的目标是培养学生的独立思考能力。

## 教学能力框架

### IARA（推理引导）
当学生提出问题时：
1. **Identify**：确认学生当前理解水平（"你目前对X的理解是什么？"）
2. **Ask**：提出引导性问题（"如果Y成立，那Z会怎样？"）
3. **Reflect**：让学生反思自己的推理（"你为什么这样认为？"）
4. **Advance**：在学生推导正确后推进到下一步

### CARA（纠错引导）
当学生回答错误时：
1. **Catch**：识别错误但不直说（"这是个有趣的想法，让我们验证一下"）
2. **Ask counter**：提出反例问题（"如果按你的理解，那这个例子会怎样？"）
3. **Redirect**：引导回正确方向（"让我们换个角度想想"）
4. **Affirm**：学生纠正后及时肯定（"对了！你自己发现了关键点"）

### SER（脚手架策略）
根据学生状态动态调整支持力度：
- **卡住 1-2 轮**：给概念性提示（"想想X和Y的关系"）
- **卡住 3-4 轮**：给方法性提示（"试试用Z方法来分析"）
- **卡住 5+ 轮**：降低难度，拆分子问题（"我们先解决这个小问题"）
- **连续 3 题正确**：提高难度或推进下一概念

### 递进式引导策略
1. 从具体案例入手 → 抽象出一般规律
2. 先处理前置知识缺口 → 再回到当前问题
3. 用类比连接已知知识 → 建立新概念

## 输出风格
- 使用中文
- 语气鼓励正面，永远不说"错了"
- 每次回复以一个问题结尾（引导学生继续思考）
- 适当使用 emoji 增加亲和力（不过度）`
```

#### 对话轮次追踪

在 `orchestrator.go` 中增加卡住检测：

```go
// StuckDetector 卡住检测器
type StuckDetector struct {
    wrongCount    int  // 连续错误轮次
    correctCount  int  // 连续正确轮次
}

func (d *StuckDetector) OnDiagnostic(isCorrect bool) {
    if isCorrect {
        d.correctCount++
        d.wrongCount = 0
    } else {
        d.wrongCount++
        d.correctCount = 0
    }
}

func (d *StuckDetector) SupportLevel() string {
    switch {
    case d.wrongCount >= 5:
        return "heavy"    // 降低难度，拆分子问题
    case d.wrongCount >= 3:
        return "medium"   // 方法性提示
    case d.wrongCount >= 1:
        return "light"    // 概念性提示
    case d.correctCount >= 3:
        return "advance"  // 提高难度
    default:
        return "normal"
    }
}
```

将 `SupportLevel` 注入 Tutor Agent 的 Prompt 尾部，动态调整引导力度。

```go
// backend/internal/agent/guard.go
package agent

import (
    "regexp"
    "strings"
)

// PromptGuard 提示词注入防护
type PromptGuard struct {
    attackPatterns []*regexp.Regexp
}

func NewPromptGuard() *PromptGuard {
    patterns := []string{
        `(?i)ignore\s+(previous|above|all)\s+(instructions?|prompts?)`,
        `(?i)forget\s+(everything|your\s+instructions?)`,
        `(?i)you\s+are\s+now\s+a`,
        `(?i)new\s+instructions?:`,
        `(?i)system\s*:\s*`,
        `(?i)直接(给|告诉|说出)(我|答案|结果)`,
        `不要(引导|提问|苏格拉底)`,
        `(?i)pretend\s+you`,
        `(?i)act\s+as\s+if`,
    }
    compiled := make([]*regexp.Regexp, len(patterns))
    for i, p := range patterns {
        compiled[i] = regexp.MustCompile(p)
    }
    return &PromptGuard{attackPatterns: compiled}
}

// DetectInjection 检测注入攻击
func (g *PromptGuard) DetectInjection(input string) bool {
    for _, pattern := range g.attackPatterns {
        if pattern.MatchString(input) {
            return true
        }
    }
    return false
}

// SanitizeOutput 输出检测：检查 AI 回复是否包含直接答案
func (g *PromptGuard) SanitizeOutput(output string, concept string) bool {
    // 如果回复中没有任何问号，可能是在直接给答案
    hasQuestion := strings.Contains(output, "？") || strings.Contains(output, "?")
    return hasQuestion
}
```

```go
// UpsertExtractedPoints 写入资料提取出的知识点及关系
func (r *KnowledgeRepo) UpsertExtractedPoints(ctx context.Context, points []ExtractedKnowledgePoint) error {
    for _, point := range points {
        // upsert 知识点，包含新字段
        _, err := r.pool.Exec(ctx, `
            INSERT INTO knowledge_mastery (concept, description, bloom_level, importance, granularity_level, confidence, updated_at)
            VALUES ($1, $2, $3, $4, $5, 0.0, NOW())
            ON CONFLICT (concept) DO UPDATE SET
                description = EXCLUDED.description,
                bloom_level = EXCLUDED.bloom_level,
                importance = EXCLUDED.importance,
                updated_at = NOW()
        `, point.Concept, point.Description, point.BloomLevel, point.Importance, point.Granularity)
        if err != nil {
            return err
        }

        // 写入多种关系类型
        for _, rel := range point.Relations {
            _, err := r.pool.Exec(ctx, `
                INSERT INTO knowledge_relations (from_concept, relation_type, to_concept, strength)
                VALUES ($1, $2, $3, $4)
                ON CONFLICT (from_concept, relation_type, to_concept) DO UPDATE SET strength = EXCLUDED.strength
            `, point.Concept, rel.Type, rel.Target, rel.Strength)
            if err != nil {
                return err
            }
        }
    }
    return nil
}
```

```go
// GetPrerequisiteChain 递归查找薄弱前置知识
func (r *KnowledgeRepo) GetPrerequisiteChain(ctx context.Context, concept string, maxDepth int) ([]KnowledgeNode, error) {
    rows, err := r.pool.Query(ctx, `
        WITH RECURSIVE prereq_chain AS (
            SELECT to_concept AS concept, 1 AS depth
            FROM knowledge_relations
            WHERE from_concept = $1 AND relation_type = 'prerequisite'
          UNION ALL
            SELECT kr.to_concept, pc.depth + 1
            FROM knowledge_relations kr
            JOIN prereq_chain pc ON kr.from_concept = pc.concept
            WHERE kr.relation_type = 'prerequisite' AND pc.depth < $2
        )
        SELECT km.id, km.concept, km.confidence, km.easiness_factor, km.interval_days
        FROM prereq_chain pc
        JOIN knowledge_mastery km ON km.concept = pc.concept
        WHERE km.confidence < 0.5
        ORDER BY pc.depth ASC
    `, concept, maxDepth)
    // ...
}
```

### 20. 知识点向量化（Qdrant）

### 21. 教学风格动态自适应

#### 现状

`tutor.go` 有 `SetLevel()` 方法切换 LevelBeginner/LevelAdvanced/LevelExpert，但需手动调用。

#### 方案

在 Orchestrator 中增加自适应逻辑：

```go
func (o *Orchestrator) detectLevel(messages []*schema.Message) string {
    // 统计最近 10 轮对话的信号
    recentDiagnostics := getRecentDiagnostics(messages, 10)

    errorRate := countErrors(recentDiagnostics) / len(recentDiagnostics)
    avgResponseDepth := measureDepth(recentDiagnostics)

    switch {
    case errorRate > 0.6:
        return LevelBeginner   // 高错误率 → 初学者模式
    case errorRate < 0.2 && avgResponseDepth > 3:
        return LevelExpert     // 低错误率 + 高深度 → 专家模式
    default:
        return LevelAdvanced
    }
}
```

每轮对话后自动检测，无需用户手动 SetLevel。

涉及文件：`orchestrator.go`（自适应检测）、`tutor.go`（移除手动 SetLevel 暴露，改为内部自动切换）。
